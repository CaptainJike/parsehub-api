# Skill Development

## Repository Skill

The project skill lives at `skills/parsehub-api`.

## Rules

- Skill names use lowercase letters, digits, and hyphens.
- `SKILL.md` frontmatter contains only `name` and `description`.
- Keep `SKILL.md` short.
- Put detailed rules in `references/`.
- Do not store secrets, production account data, or environment-specific credentials in skills.
- Update `AGENTS.md` when adding new skill references.

## Reference Files

- `architecture-rules.md`: module and boundary rules.
- `api-rules.md`: response and versioning rules.
- `provider-rules.md`: parser/provider rules.
- `security-rules.md`: auth, secret, URL, and payment safety.
- `testing-rules.md`: required test coverage.
